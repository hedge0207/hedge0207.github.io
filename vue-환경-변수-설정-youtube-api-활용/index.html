Vue 환경 변수 설정(Youtube api 활용)

10.01.2021 — Vue — 1 min read

  • 유튜브에서 api key와 api url을 받아서 직접 만든 페이지에 유튜브 동영상이 뜨게 하는 것이 목적

  • 환경변수 관리(.env)

    • 뷰 CLI로 생성하여 애플리케이션을 제작하게 되면 기본적으로 웹팩을 사용한다. 따라서, 모드에 따라 웹팩의 설정이 달라진다.

    • Vue CLI는 3가지 모드를 가지고 있다.

      • development: 개발 용도, npm run serve 명령어로 실행할 때의 모드
      • production: 배포 용도, npm run build 명령어로 실행할 때의 모드
      • test: 테스트 용도, npm run test:unit 명령어로 실행할 때의 모드

  • 위의 3가지 모드 이외에 직접 모드를 커스텀 해서 사용할 수 있다.

    • 정의
    1//package.json
    2
    3"scripts": {
    4  //기존의 3가지 모드
    5  "serve": "vue-cli-service serve",
    6  "build": "vue-cli-service build",
    7  "lint": "vue-cli-service lint",
    8  
    9  //직접 커스텀한 모드
    10  "goguma":"vue-cli-service serve --mode goguma"
    11  //vue-cli-service serve까지는 그냥 입력하면 되며 --mode goguma는 goguma 모드로 사용하겠다는 뜻이다.
    12},
    • 실행
    1#아래와 같이 실행하면 goguma 모드로 실행된다.
    2$npm run goguma

  • CLI에서는 프로젝트 루트 폴더에 아래와 같은 규칙으로 환경 변수 파일을 생성할 수 있다.

    • .env: 모든 모드에 적용되는 환경 변수 파일
    • .env.local: 모든 모드에 적용되나 개인만 사용하는 파일
    • .env.[mode]: 특정 모드에 해당하는 환경 변수 파일
    • .env.[mode].local: 특정 모드에 해당하지만 개인이 사용하는 파일

  • 예시

    • NODE_ENV: 애플리케이션 모드를 가리키는 변수(development, production, test)
    • BASE_URL: vue.config.js 파일에 정의된 publicPath의 값과 동일한 변수
    1//.env.development
    2
    3VUE_APP_ANIMAL="cat"
    1.env.goguma
    2
    3NODE_ENV="goguma"
    4VUE_APP_ANIMAL="dog"
    • 위와 같이 각기 다른 모드에 대한 환경 변수 파일이 있을 때 npm run serve로 실행하면 아래와 같은 결과가 나온다.
    1console.log(env.proecess.VUE_APP_ANIMAL)
    2
    3//cat
    • npm run goguma로 실행하면 아래와 같은 결과가 나온다
    1console.log(env.proecess.VUE_APP_ANIMAL)
    2
    3//dog

  • 각 환경 변수 파일의 우선 순위는 모드에 따라 달라진다.

    • 예를 들어 배포 환경일 때는 .env.production의 우선권이 .env.development의 우선권 보다 높고, 개발 환경일 때는 그 반대이다.

  • API KEY는 관리를 철저히 해야 한다. github에 올라가지 않도록 gitignore로 관리를 해줘야 한다.

    • .gitignore에 최초 생성시에 .env.local파일이 포함되어 생성된다.

    • .env.local파일에 API KEY를 관리하면 된다.

    • 프로젝트 폴더 내부(최상위 폴더)에 .env.local파일 생성 후 API KEY를 입력한다.

      1VUE_APP_API_KEY=AbCdEfGhIjK7LMNop9QrSTU1VWx7Yz
      2
      3-시작은 반드시 VUE_APP_로 시작해야 하며 = 앞뒤에 공백이 있어선 안된다.

    • API KEY를 입력했던 파일에 아래와 같이 입력한다.

      1const API_KEY = process.env.VUE_APP_API_KEY

    • 위 과정을 거친 후 serve를 한번 껐다 켜야한다.

    • 이렇게 한다고 완전히 숨길 수 있는 것은 아니지만 적어도 github에서는 숨길 수 있다.

  • 코드

    • App.vue
    1<template>
    2  <div id="app" class="container">
    3      <!--search-videos이벤트가 발생하면 searchVideos메소드를 실행-->
    4      <SearchBar @search-videos="searchVideos"/>
    5    <div class="row">
    6      <!--selectedVideoId, selectedVideoTitle을 자식 컴포넌트에게 넘겨준다.-->
    7      <YoutubeDetail :selectedVideoId="selectedVideoId" :selectedVideoTitle="selectedVideoTitle"/>
    8      <!--영상 리스트를 띄워주는 컴포넌트-->
    9      <YoutubeList :videos="videos" @select-video="selectVideo"/>
    10    </div>
    11  </div>
    12</template>
    13
    14<script>
    15//$npm i axios를 통해 axios를 설치 한 후 import
    16import axios from 'axios'
    17
    18//App.vue에서 사용할 컴포넌트들을 import
    19import SearchBar from './components/SearchBar'
    20import YoutubeList from './components/YoutubeList'
    21import YoutubeDetail from './components/YoutubeDetail'
    22
    23//google 클라우드 플랫폼-새 프로젝트 생성-좌측 상단 햄버거-API 및 서비스-대시보드-
    24//-API및 서비스 사용 설정-YouTube Data API v3-사용 설정-사용자 인증 정보 만들기-
    25//-API선택, API를 호출할 위치는 웹브라우저(자바스크립트)선택, 엑세스할 데이터는 공개 데이터 선택
    26//-어떤 사용자 인증 정보가 필요한가요? 클릭하면 API KEY를 발급 받을 수 있다.
    27//발급 받은 이후부터는 사용자 인증 정보 탭에서 확인 할 수 있다.
    28const API_KEY = process.env.VUE_APP_API_KEY
    29
    30//https://developers.google.com/youtube/v3/docs/search/list에 접속하면 HTTP 요청 url이 있다.
    31const API_URL = 'https://www.googleapis.com/youtube/v3/search'
    32
    33
    34export default {
    35  name: 'App',
    36  components:{
    37    SearchBar,
    38    YoutubeList,
    39    YoutubeDetail,
    40  },
    41  data: function(){
    42    return {
    43      videos: [],
    44      selectedVideoId: '',
    45      selectedVideoTitle:'',
    46    }
    47  },
    48  methods: {
    49    searchVideos: function(keyword){
    50      // axios를 통해 요청을 보내서 videos를 변경
    51      // 아래 경로는 관리자 도구-network-preview에서 확인 할 수 있다. 응답으로 json 오브젝트가 온다.
    52      // params 오브젝트에 들어가는 실제 내용은 아래와 같은 쿼리 스트링이다.
    53      //'?KEY=AbCdEfGhIjK7LMNop9QrSTU1VWx7Yz&part=video&part=snippet&q=임의의 값'
    54      axios.get(API_URL, {
    55        params:{
    56          key: API_KEY,
    57          type: 'video',
    58          part: 'snippet',
    59          q:keyword,
    60        },
    61        //화살표 함수 사용
    62        //응답으로 온 json 오브젝트의 response.data.items에 요청 받은 내용이 들어 있다.
    63      }).then((response)=>{
    64        this.videos = response.data.items
    65        //요청 받은 내용이 오지 않았을 경우 콘솔 창에 error를 출력
    66      }).catch((error)=>{
    67        console.error(error)
    68      })
    69    },
    70    selectVideo:function(videoId, videoTitle){
    71      this.selectedVideoId=videoId
    72      this.selectedVideoTitle=videoTitle
    73    },
    74  }
    75}
    76</script>
    77

    • SearchBar.vue
    1<template>
    2  <div class="row">
    3      <!--입력 받을 keyword 데이터에 v-model 설정, enter가 입력되면 searchVideos함수 실행-->
    4      <input type="text" v-model="keyword" @keypress.enter="searchVideos" placeholder="동영상을 검색하세요">
    5  </div>
    6</template>
    7  
    8<script>
    9export default {
    10    name:'SearchBar',
    11   data:function(){
    12      return {
    13        keyword: '',
    14      }
    15    },
    16    methods: {
    17      searchVideos:function(){
    18        // trim은 좌우측 공백을 제거하는 함수, 만일 공백을 제거했을 때 빈 문자열이라면 입력된 것이 없다는 뜻이므로 알림창을 띄운다.
    19        if (!this.keyword.trim()){
    20          alert('검색어를 입력해 주세요.')
    21        }else{
    22          //공백이 아니라면 emit을 실행
    23          //search-videos이벤트를 실행하고 keyword.trim()을 인자로 넘긴다.
    24          this.$emit('search-videos', this.keyword.trim())
    25        }
    26      },
    27    },
    28}
    29</script>
    30  
    31<style>
    32  input{
    33    width:75%;
    34    margin-bottom: 10px;
    35  }
    36</style>

    1<template>
    2<ul class="list-group col-8">
    3      <!--v-for를 사용할 경우 key값을 줘야 하는데 video.etag가 유일한 값을 가지므로 key의 역할을 할 수 있다.-->
    4      <!--select-video 이벤트가 실행되면 selectVideo 메소드 실행-->

    23            //select-video는 위에서도 썼는데 자식 컴포넌트와 부모 컴포넌트의 이벤트명이 겹쳐도 상관 없다.
    24            this.$emit('select-video',videoId,videoTitle)
    25        },
    26    },
    27}

    • YoutubeListItem.vue
    1<template>
    2  <li class="list-group-item" @click="selectVideo">
    3    <!--
    4      :src="video.snippet.thumbnails.default.url"로 써도 되지만 길어지므로 아래에 computed에 함수를 선언해서 짧게 써준다.
    5      videoTitle, videoDescription도 마찬가지로 그냥 video.snippet.title, video.snippet.description로 써도 되지만 짧게 써주기 위해 computed에 써준다.
    6    -->
    7    <div class="media">
    8      <img :src="thumbnailUrl" :alt="thumbnailUrl" class="mr-3">
    9        <div class="media-body">

    11          <p>{{ videoDescription }}</p>
    12        </div>
    13    </div>
    14  </li>
    15</template>
    16
    17<script>
    18export default {
    19    name:'YoutubeListItem',
    20    // 부모 컴포넌트에서 받은 videos를 props에 저장
    21    props: {
    22      video: Object,
    23    },
    24    computed: {
    25      thumbnailUrl: function () {
    26        return this.video.snippet.thumbnails.default.url
    27      },

    35    methods:{
    36      selectVideo: function () {
    37        //아래 video.id.videoId는 개발자도구-network-preview에서 볼수 있다.
    38        this.$emit('select-video',this.video.id.videoId, this.videoTitle)
    39      },
    40    },
    41}
    42</script>

    • YoutubeDetail.vue
    1<template>
    2  <div class="col-4 align-center">
    3      <!--
    4        영상을 클릭하지 않았을 때도 기본 영상이 뜨는 것을 막기 위해 v-if를 사용, 
    5        selectedVideoId가 지정되지 않았다면 빈 문자열일 것이므로 false가 되어 영상이 뜨지 않는다.

    7        iframe은 bootstrap-embeds에 있는 태그이다. iframeUrl은 굳이 함수로 선언하지 않고

    10      -->
    11      <iframe 
    12        v-if="selectedVideoId" 
    13        :src="iframeUrl"
    14      ></iframe>
    15      <!--

    17        v-html에 보안이슈가 있긴 하지만 아래의 경우 보안에 문제 될 것이 없다는 것이 확실하므로 사용한다.
    18      -->
    19      <h4 class="mt-0 mb-1" v-html="selectedVideoTitle"></h4>      
    20  </div>
    21</template>

    29      selectedVideoTitle:String,

    31    computed:{
